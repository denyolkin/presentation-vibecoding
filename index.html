The problem isn't time management. It's cognitive load.

I know GTD. I've read Dorofeev. I use Asana. I have a calendar. I'm not disorganized.

But: 200 emails across 3 inboxes. 10 people who each need something different. 5 projects running in parallel. Meetings back-to-back. Slack pinging. Invoices piling up.

The frameworks I studied — Dorofeev, 4DX, Premortem — I knew them. I just couldn't apply them in the moment. Too many inputs. Too fast. The strategic thinking drowns in operational noise.

I needed something that remembers what I forget. That applies the right framework at the right time. That doesn't get tired.

CLAUDE.md — the brain

Not a config file. An operating philosophy. Defines personality, security rules, thinking patterns, content style, task methodology, quality standards. The same file works across Telegram bot, Claude Code, Cursor, and Dashboard.

Every plan gets attacked 3 times before I see it

I told the AI: "Before showing me any plan, critique it yourself in 3 rounds. Different role each time. I only see the final version after all 3 attacks."

Round 1
Senior Software Architect

Attacks: API contracts, data flow, race conditions, backward compatibility, deploy order, performance bottlenecks.

Round 2
Senior Product Manager

Attacks: Does it solve the problem? Simplest solution? Scope creep? User experience? Is it testable?

Round 3
Domain Stakeholder

Role depends on context: accountant for finance features, team member for team features, attacker for security. Different every time.

If a round finds zero real issues (not cosmetic), remaining rounds are skipped.

Stress-test: every output gets attacked too

Not just plans — every proposal, diagnosis, strategy, analysis, code. Three rounds: propose → find real flaws → fix. Key rule: testable claims = actually test them (run curl, grep, read docs, execute code). Reasoning about testable facts is lazy.

This applies silently by default. I can also invoke it explicitly with "/stress-test" to see all 3 rounds.

Security: Auth Proxy on VPC

All API keys (Gmail, Asana, Slack, Google, tl;dv, Ultrahuman) sit behind a dedicated auth proxy on a private VPC network. The AI never sees raw credentials. Two servers: bot server (public-facing) and auth proxy (VPC only, no public access). Every tool call goes through the proxy with a Bearer token.

Why not just put keys in .env? Because the bot processes user messages, and prompt injection is real. The proxy validates every request before forwarding.

Tests, code review, deploy safety

Unit tests + edge cases

Every non-trivial change gets a test strategy (testing-methodology skill). The AI writes tests, runs them, fixes failures before showing me the result.

Multi-role code review

Before deploy: code gets reviewed as senior dev (correctness), security auditor (vulnerabilities), and PM (does it solve the right problem).

Deploy skill

Never manual git push. A /deploy skill runs: lint, tests, review, commit, push, SSH to server, restart service. One command.

19 background schedulers

Email checks (3 accounts), Asana inbox, Slack mentions, meeting transcripts, task reminders, morning briefing, marketing reports, health checks, cache refresh, PR notifications, proactive alerts (stale tasks, calendar overload).

Bonus: synthetic market research

One of the 20 skills. LLM generates demographic personas matching target market, role-plays their reaction to a product concept, then scores responses using Semantic Similarity Rating. 90% correlation with real survey data (validated by PyMC Labs across 57 consumer surveys). $0 per respondent. Results in minutes, not weeks.

Three modes: interactive (step by step), quick (5 personas, instant), comparative (A/B test variants).

Thinking Partner

The AI doesn't give answers. It asks the right questions. Applies frameworks I studied but forgot in the daily grind.

Rule #1: "INTERVIEW BEFORE OUTPUT — never give recommendations until you've interviewed Denis. Ask ONE question at a time."

11 thinking frameworks, loaded from one file

Dorofeev "Jedi Techniques"

"What's the first physical action?" Tasks start with a verb. Banned: "think about", "analyze", "figure out". WIP limit: 5 active tasks.

4DX (McChesney)

One WIG: "From X to Y by When." Lead measures (actions I control weekly) vs. lag measures (results). Whirlwind = everything else.

Descartes Square

4 quadrants: what happens if I do / don't? What WON'T happen if I do / don't? Breaks binary paralysis.

Stakeholder Simulation

AI role-plays my CEO, a team member, a partner — predicts their reaction before I act.

Board Simulation

3-4 personas debate a decision from different angles. I watch, pick strongest arguments.

Strategic Deep Dive (CRIT)

Big decisions: structured walk-through of context, risks, implications, tradeoffs.

Second-Order Consequences

What happens after the thing happens? Ripple effects beyond the obvious first outcome.

Competitive/Adversarial

Tests strategy from competitor's eyes. "What would they do if they saw this?"

Risk Identification

Probability × severity × reversibility matrix. Forces ranking, not just listing.

Vital 20% Audit

When overwhelmed: which 20% of activities drive 80% of results? Cut everything else.

Decision Forcing

Breaks analysis paralysis. "You must decide by end of this conversation."

Where the frameworks come from

Maxim Dorofeev — "Jedi Techniques" (Джедайские техники), 7th ed., 2025, MIF. Task clarity, monkey-readable actions, WIP limits. Built on Kahneman, David Allen, Goldratt, Taleb.
McChesney, Covey, Huling — "The 4 Disciplines of Execution". WIG, lead measures, scoreboard, cadence of accountability.
Geoff Woods — "The AI-Driven Leader". The idea that AI should be a thinking partner, not a task executor. Integrated Feb 2026.
James Clear — "Atomic Habits". Identity-based habits, 1% improvement loops.
